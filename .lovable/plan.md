
# План: Добавить выбор USDT/USDC в виджет покупки на Dashboard

## Проблема

В Dashboard используется компонент `PurchaseWidget`, который:
- Показывает только выбор сети (ETH/BNB/SOL)
- Не имеет выбора способа оплаты (USDT/USDC)
- Не интегрирован с логикой покупки (`usePurchase` hook)
- Не сохраняет транзакции в базу данных

В то время как `PresaleCard` на главной странице уже полностью функционален.

## Решение

Обновить `PurchaseWidget` для Dashboard, добавив:

1. **Выбор способа оплаты** — кнопки Native/USDT/USDC (как в PresaleCard)
2. **Интеграция с `usePurchase` hook** — реальная логика покупки
3. **Сохранение транзакций** — через `useTransactions` hook
4. **Корректный расчёт токенов** — учитывая курс ETH/BNB и стейблкоинов
5. **Состояния загрузки и ошибок** — как в PresaleCard

## Визуальная структура

```text
┌─────────────────────────────────────┐

├─────────────────────────────────────┤

├─────────────────────────────────────┤
│ [        Buy Now         ]          │
│ Min: $10 • Max: $50,000             │
└─────────────────────────────────────┘
```

## Технические изменения

### Файл: `src/components/dashboard/PurchaseWidget.tsx`

**Что изменится:**

1. Импорты — добавить wagmi hooks, usePurchase, useTransactions
2. Удалить SOL из списка сетей (не поддерживается пресейлом)
3. Добавить состояние `selectedPayment` для выбора Native/USDT/USDC
4. Интегрировать `usePurchase` hook с обработкой всех состояний
5. Добавить `useTransactions` для сохранения транзакций
6. Обновить расчёт токенов с учётом курса валют
7. Добавить UI для состояний loading/success/error

## Результат

- Пользователь сможет выбрать способ оплаты (ETH/BNB/USDT/USDC) прямо в Dashboard
- Все покупки будут корректно обрабатываться через `usePurchase`
- Транзакции будут автоматически сохраняться в историю
